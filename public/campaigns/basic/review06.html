簡単な経歴

自社開発企業でPMや上流工程の仕事を経て、現在は受託開発企業でエンジニアとして働いています。

段階的なカリキュラムと手厚いサポートで、ちゃんと開発できる基礎が身についた

はじめまして。フィヨルドブートキャンプ卒業生の maimu です。

私は自社開発企業で PM や開発の上流工程を担当しながらエンジニアへのキャリアチェンジを目指してフィヨルドブートキャンプで学習をしていました。現在は受託開発企業で働いています。

一人の卒業生として、私が「フィヨルドブートキャンプを選んでよかった！」と感じる理由を２つご紹介したいと思います。

段階的に学べるカリキュラム

１つ目はカリキュラムの構成についてです。

フィヨルドブートキャンプの Web サイトを見ると受講できるコースとその内容を確認することができます。私が受講していた「Rails エンジニアコース」を見てみると、Rails を学ぶ手前で Vi、 Linux、 Git、 GitHub、 データベースなどを学ぶプラクティスが含まれています。

私はフィヨルドブートキャンプに入学する前に独学で Ruby と Rails を学んでいた期間があるのですが、それ以前の開発に必要な基礎となる Linux や Git を学ぶステップを飛ばして特定のプログラミング言語を学ぼうとしたことで最終的にどのような流れで学んでいけばよいのかがわからなくなり挫折した経験があります。この経験があったため、フィヨルドブートキャンプが公開しているカリキュラムの構成を見て、卒業すると「ちゃんと開発をできる基礎を得ることができそう」と感じたことを覚えています。また、実際に卒業して就職してみて、当時感じたことは事実だったなと実感しています。

各カリキュラムは参考書籍を読むステップや実際に手を動かして学ぶステップとが組み合わされているため、「知って、試して、理解する」サイクルをうまく回すことができます。また、一つのカリキュラムを終えて次のカリキュラムに進むと、以前学んだことを活かしながら手を動かせるようにカリキュラム同士が繋がっていて、基礎を積み上げながら学習を進めていくことができます。

各カリキュラムが開発に必要なレイヤーごとにパートが分かれていたことで、それぞれで何をするのか・できるのかを段階を踏みながら理解を深めていくことができたなと感じています。在学中は卒業と就職を目指して必死だったため、あまり実感できていなかったのですが、卒業後に実際に Web サービスの開発に携わるようになると、「今自分はどこのレイヤーを扱っているのか」を認識して課題と向き合うことの大切さをとても強く感じるようになりました。Web サービスをメンテナンスして拡張していくとなるとフレームワークやその周辺技術への理解は欠かせません。正直なところ、今でもレイヤーが混ざってしまって課題をうまく扱えないときは多いのですが、自分がわかっていないことを分解していく作業ではフィヨルドブートキャンプ時代の個々のカリキュラムで作られた知識の土台が自分を支えて助けてくれています。

充実したサポート体制

２つ目はサポート体制に関してです。

カリキュラムの課題をレビューしてくださるメンター陣の方々の心強いサポートには何度も助けられました。カリキュラムの学習は基本自分一人で進めていきますが、一人で学ぶことは自分が独学で挫折したように迷いやすく大変な作業です。提出した課題に対してメンター陣の方々からフィードバックやヒントをいただけることで、課題の内容について自分には見えていなかった側面に気づいて考えるべき観点が広がっていったり、別の効果としてプルリクエスト上でのコミュニケーションのとり方を身に着けていくことができました。また、ペアプロの時間を取っていただき、画面を共有しながらエディタやターミナルの操作を教えていただくこともあり、非同期のコミュニケーションだけでは学ぶことができないテクニック的な部分も知ることができました。技術面のサポートだけでなく、就職に関するサポートも受講生一人ひとりの話をしっかり聞いてくださり、自分が目指す方向性を見極めながら取り組むことができました。

サポート体制ではありませんが、フィヨルドブートキャンプは受講生同士のコミュニケーションも活発で輪読会や勉強会もたくさん開催されています。こういった会に参加をすることで一人で頑張っているわけではないんだと励みをもらったり、メンター陣の方々と同様に自分にはない考え方や情報を得られたりしました。

以上が私が「フィヨルドブートキャンプを選んでよかった！」ことになります。
